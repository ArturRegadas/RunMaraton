Importador de Casos de Teste

Selecione um arquivo ZIP contendo a pasta AAAA (representando o ano ou identificador).

O sistema detecta automaticamente:

  • O ano ou identificador da competição.
  • Se os exercícios pertencem à Fase Final ou Fase Local.
  • As letras dos exercícios.

Estrutura esperada

O ZIP deve conter exatamente uma pasta principal com 4 caracteres:

2024/

Dentro dela podem existir as pastas:

2024/
├── Fase Final/
└── Fase Local/

Ou, caso não existam, todos os exercícios serão considerados da Fase Final:

2024/
├── A-matematica.zip
├── B-grafos.zip
└── C-geometria.zip

Estrutura de cada exercício

A-matematica.zip
├── description/
│   ├── problem.info
│   └── INTERIF2024.pdf
├── input/
├── output/
└── limits/
    └── c

Convenções de nomenclatura

Pasta principal

O arquivo enviado deve ser um ZIP contendo exatamente uma pasta principal no formato:

AAAA

Onde AAAA representa obrigatoriamente um identificador de exatamente 4 caracteres:

  • Para problemas do INTERIF: deve ser um ano composto por 4 algarismos (ex: 2024).
  • Para problemas que não são do INTERIF: pode ser qualquer combinação de 4 letras ou algarismos (ex: MARS, 0000, TEST).

Exemplos de pastas válidas:

2024
MARS
0000

Pastas de fase

Opcionalmente podem existir as seguintes pastas:

Fase Final
Fase Local

Os nomes devem ser exatamente esses.

Se nenhuma delas existir, todos os exercícios serão considerados pertencentes à Fase Final.

ZIPs dos exercícios

Cada exercício deve estar em um arquivo ZIP separado e deve seguir OBRIGATORIAMENTE o seguinte formato:

{LETRA_MAIUSCULA}-nome.zip

Exemplos válidos:

A-matematica.zip
B-grafos.zip
C-exercicio1.zip
D-problema-de-teste.zip

Exemplos inválidos (serão ignorados pelo importador):

a-matematica.zip (letra minúscula)
A_matematica.zip (usa underline em vez de hífen)
A.zip (falta o hífen e o nome)
ProblemaA.zip (formato incorreto)

O importador utiliza apenas a letra antes do primeiro hífen para identificar o exercício e o restante para identificar o nome/slug.

Pastas obrigatórias dentro do ZIP do exercício

description
input
output
limits

Os nomes devem ser exatamente esses (inclusive letras minúsculas).

Arquivo problem.info

Deve existir exatamente neste local:

description/problem.info

Conteúdo esperado:

basename=matematica
fullname=Matemática
descfile=INTERIF2024.pdf
Campo Obrigatório Descrição
basename Sim Nome interno do problema (não é utilizado pelo importador).
fullname Sim Nome completo do problema. Será gravado em src/info.txt.
descfile Sim Nome do PDF existente dentro da pasta description.

Arquivo PDF

O PDF pode possuir qualquer nome, porém ele deve ser exatamente igual ao valor informado em descfile.

Exemplo:

description/
├── problem.info
└── INTERIF2024.pdf

Pasta limits

Deve existir exatamente:

limits/
└── c

O arquivo deve se chamar exatamente c.

Conteúdo esperado:

echo 1
echo 1
echo 512
echo 55
exit 0

Os quatro valores representam:

  1. Tempo limite (segundos)
  2. Número de repetições
  3. Limite de memória (MB)
  4. Limite de saída (KB)

Pasta input

Deve conter todos os casos de entrada.

Os arquivos podem possuir qualquer nome.

Exemplos:

input/
├── 1
├── 2
├── 3
└── 4

ou

input/
├── 001.in
├── 002.in
└── 003.in

Pasta output

Deve conter as respostas correspondentes aos arquivos presentes em input.

Os nomes devem corresponder aos arquivos de entrada.

Exemplos:

output/
├── 1
├── 2
├── 3
└── 4

ou

output/
├── 001.out
├── 002.out
└── 003.out

Arquivos gerados pelo importador

Após a importação, o sistema criará automaticamente:

static/
└── ANOFLETRA.pdf

problems/
└── ANOFLETRA/
    ├── inputs/
    ├── outputs/
    └── limits.json

src/
└── info.txt

Exemplo:

static/
└── 2024FA.pdf

problems/
└── 2024FA/
    ├── inputs/
    ├── outputs/
    └── limits.json

src/info.txt

Onde:

  • 2024 / AAAA → ano ou identificador da competição.
  • F → Fase Final.
  • L → Fase Local.
  • A → letra do exercício.

O arquivo limits.json terá o formato:

{
    "time": "1",
    "number_of_repetitions": "1",
    "memory_limit_MB": "512",
    "output_limit_KB": "55"
}

O arquivo src/info.txt receberá uma linha para cada exercício:

2024FA = Matemática
MARSFA = Nome do Exercício

Observações Importantes

  • Todos os nomes de pastas e arquivos obrigatórios diferenciam letras maiúsculas e minúsculas.
  • Arquivos ou pastas com nomes diferentes dos especificados poderão impedir a importação do exercício.
  • Os arquivos da pasta input e da pasta output são copiados exatamente como estão, sem renomeação.
  • Se algum exercício estiver incompleto ou inválido, apenas ele será ignorado; os demais continuarão sendo importados normalmente.
